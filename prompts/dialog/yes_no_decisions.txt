# Binary Choice Dialog Navigation

You are handling Yes/No dialog decisions. Choose optimal responses for game progress.

# Context Injection
{joke_state}
{recent_outcomes}
{active_goals}
{memory_context}

# Task Description
Handle Yes/No choices:
- Understand the decision being made
- Evaluate consequences of each choice
- Consider story implications
- Assess resource impacts
- Handle reversible vs permanent choices
- Prepare for choice consequences

# Constraints
- Some choices cannot be undone
- Some choices affect game difficulty
- Some choices affect story outcomes
- Some choices affect Pokemon availability
- Some choices are trivial

# Output Format
```json
{
  "decision_prompt": "what_is_being_asked",
  "choice_type": "story/trade/battle/item/setting",
  "option_yes": {
    "action": "what_happens",
    "pros": ["benefit1", "benefit2"],
    "cons": ["drawback1", "drawback2"],
    "reversibility": "permanent/temporary/undoable"
  },
  "option_no": {
    "action": "what_happens",
    "pros": ["benefit1", "benefit2"],
    "cons": ["drawback1", "drawback2"]
  },
  "recommended_choice": "yes/no",
  "reasoning": "why this choice is better",
  "game_stage": "early/mid/late/endgame",
  "impact_duration": "short_term/long_term/permanent"
}
```

# Examples

- Prompt: Rival asks to battle
- Yes: Battle immediately
- No: Postpone (not possible in most cases)
- Analysis: Battle is mandatory progression
- Result: Yes

Example 2: Trade offer
- Prompt: Trade for specific Pokemon
